# CafeOps Dashboard PRD

## 1. 프로젝트 개요

CafeOps Dashboard는 카페 운영자가 재고 상태, 발주 필요 품목, 판매 흐름을 한 화면에서 확인할 수 있도록 돕는 운영 대시보드입니다.

카페 매니저 경험에서 재고 확인, 발주 판단, 판매 흐름 확인이 반복적으로 발생한다는 점에 주목했습니다. 이 프로젝트는 운영자가 현재 매장 상황을 빠르게 파악하고, 기준에 따라 재고와 발주 여부를 판단할 수 있도록 돕는 것을 목표로 합니다.

## 2. 문제 정의

카페 운영에서는 품목 수가 많아질수록 현재 재고가 충분한지, 어떤 품목을 먼저 발주해야 하는지 빠르게 판단하기 어렵습니다.

단순 재고 목록만으로는 운영자가 우선순위를 정하기 어렵기 때문에, 현재 재고와 안전 재고를 비교해 재고 상태를 자동 분류하고, 발주가 필요한 품목을 명확하게 보여주는 구조가 필요합니다.

## 3. 목표

- 재고 상태를 부족 / 정상 / 과잉으로 자동 분류합니다.
- 발주가 필요한 품목을 빠르게 확인할 수 있게 합니다.
- 운영자가 상품을 등록하고 수정할 수 있게 합니다.
- 판매 흐름과 재고 부족 현황을 요약 카드와 차트로 확인할 수 있게 합니다.
- 주요 계산 로직은 테스트 가능한 함수로 분리합니다.

## 4. 주요 사용자

- 카페 매니저
- 매장 운영 담당자
- 재고 및 발주 담당자

## 5. MVP 범위

### 포함

- 재고 목록 조회
- 검색, 카테고리 필터, 정렬
- 재고 상태 자동 분류
- 권장 발주 수량 계산
- 상품 등록 / 수정 폼
- Zod 기반 폼 검증
- 요약 카드
- 차트 1~2개
- MSW 기반 mock API
- Vitest 기반 계산 로직 테스트

### 제외

- 로그인
- 실제 DB 연동
- 권한 관리
- 엑셀 다운로드
- 다중 매장 관리
- 실시간 알림
- 결제/주문 연동

## 6. 핵심 사용자 흐름

1. 운영자는 대시보드에서 오늘의 판매 요약과 재고 부족 현황을 확인합니다.
2. 재고 관리 페이지에서 전체 품목을 검색, 필터, 정렬합니다.
3. 현재 재고와 안전 재고를 기준으로 부족 품목을 확인합니다.
4. 부족 품목의 권장 발주 수량을 확인합니다.
5. 필요한 경우 상품 정보를 등록하거나 수정합니다.

## 7. 성공 기준

- 사용자는 재고 부족 품목을 빠르게 파악할 수 있습니다.
- 재고 상태는 현재 재고와 안전 재고 기준으로 자동 분류됩니다.
- 상품 등록/수정 시 잘못된 입력은 폼 단계에서 차단됩니다.
- 재고 상태 계산 로직과 권장 발주 수량 계산 로직은 테스트로 검증됩니다.
- mock API 기반으로 백엔드 없이 주요 화면 상태를 확인할 수 있습니다.
