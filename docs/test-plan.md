# CafeOps Dashboard 테스트 및 검증 기준

## 1. 목적

CafeOps Dashboard의 핵심 계산 로직과 주요 사용자 흐름이 문서에 정의한 기준대로 동작하는지 검증한다.

특히 재고 상태와 권장 발주 수량은 운영 판단에 직접 영향을 주는 값이므로 UI와 분리해 테스트 가능한 함수로 검증한다.

---

## 2. 테스트 범위

| 구분           | 대상                              | 도구                                 |
| -------------- | --------------------------------- | ------------------------------------ |
| 단위 테스트    | 재고 계산 함수                    | Vitest                               |
| 스키마 검증    | 상품 등록/수정 폼 schema          | Vitest + Zod                         |
| API 흐름 검증  | Mock API 요청/응답                | MSW                                  |
| 화면 상태 검증 | Loading / Empty / Error / Success | 수동 확인 또는 React Testing Library |
| 빌드 검증      | 운영 빌드 성공 여부               | npm run build                        |

---

## 3. 단위 테스트 대상

| 함수                                  | 검증 내용                                                             |
| ------------------------------------- | --------------------------------------------------------------------- |
| getDaysSinceLastChecked               | 마지막 재고 확인일 이후 지난 일수 계산                                |
| getDaysUntilNextCycleRestock          | 다음 보충 가능일까지 남은 일수 계산                                   |
| getEstimatedCurrentStock              | 마지막 확인 재고, 입고 수량, 예상 사용량을 반영한 현재 예상 재고 계산 |
| getRequiredStockUntilNextCycleRestock | 다음 보충 주기까지 필요한 기준 재고 계산                              |
| getRecommendedOrderQuantity           | 권장 발주 수량 계산                                                   |
| getInventoryStatus                    | 부족 / 정상 / 과잉 상태 분류                                          |

## 4. 계산 로직 검증 기준

### 현재 예상 재고

| 조건                                                              | 기대 결과                         |
| ----------------------------------------------------------------- | --------------------------------- |
| 마지막 확인 재고 30, 입고 수량 0, 예상 하루 사용량 5, 지난 일수 3 | 15                                |
| 마지막 확인 재고 5, 입고 수량 0, 예상 하루 사용량 4, 지난 일수 3  | 0                                 |
| 지난 일수가 0                                                     | 마지막 확인 재고 + 입고 수량 반환 |

### 다음 보충 주기까지 필요한 기준 재고

| 조건                                                      | 기대 결과 |
| --------------------------------------------------------- | --------- |
| 다음 보충 가능일까지 8일, 예상 하루 사용량 5, 버퍼 재고 5 | 45        |
| 다음 보충 가능일까지 0일, 예상 하루 사용량 5, 버퍼 재고 5 | 5         |
| 예상 하루 사용량 0, 버퍼 재고 5                           | 5         |

### 권장 발주 수량

| 조건                                 | 기대 결과 |
| ------------------------------------ | --------- |
| 필요 기준 재고 45, 현재 예상 재고 15 | 30        |
| 필요 기준 재고 45, 현재 예상 재고 50 | 0         |

### 재고 상태

| 조건                                                 | 기대 결과 |
| ---------------------------------------------------- | --------- |
| 현재 예상 재고 > 최대 재고                           | overstock |
| 현재 예상 재고 < 다음 보충 주기까지 필요한 기준 재고 | low       |
| 현재 예상 재고가 기준 재고 이상이고 최대 재고 이하   | normal    |

## 6. Mock API 검증 기준

| API                       | 검증 내용                              |
| ------------------------- | -------------------------------------- |
| GET /inventory            | 재고 목록을 반환한다                   |
| POST /inventory           | 새 재고 품목을 추가한다                |
| PATCH /inventory/:id      | 기존 재고 품목을 수정한다              |
| GET /sales/summary        | 요약 카드 데이터를 반환한다            |
| GET /sales/trend          | 매출 추이 차트 데이터를 반환한다       |
| GET /sales/category-ratio | 카테고리별 판매 비중 데이터를 반환한다 |

---

## 7. 화면 상태 검증 기준

| 상태    | 검증 내용                                       |
| ------- | ----------------------------------------------- |
| Loading | 데이터를 불러오는 동안 로딩 UI 표시             |
| Empty   | 재고 목록 또는 검색 결과가 없을 때 빈 상태 표시 |
| Error   | API 요청 실패 시 에러 상태 표시                 |
| Success | 데이터 조회 성공 시 테이블/카드/차트 표시       |

---

## 8. 최종 검증 기준

- `npm run dev`가 정상 실행된다.
- `npm run build`가 정상 통과한다.
- Vitest 설정 이후 `npm run test`가 정상 통과한다.
- 재고 계산 기준이 PRD, 기능 명세, 데이터 모델, 코드에서 일치한다.
- 주요 사용자 흐름이 README에 정리되어 있다.
