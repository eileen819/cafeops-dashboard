| 수정일                              | 마지막 수정일                                            |

### 2.5 현재 예상 재고 계산

현재 예상 재고는 마지막 확인 재고에서 마지막 확인일 이후 예상 사용량을 차감하고, 마지막 확인 이후 입고된 수량을 더해 계산한다.

#### 계산에 사용하는 값

| 항목                       | 설명                              |
| -------------------------- | --------------------------------- |
| 마지막 확인 재고           | 마지막으로 확인한 재고 수량       |
| 마지막 재고 확인일         | 재고 수량을 확인한 날짜           |
| 마지막 확인 이후 입고 수량 | 마지막 재고 확인 이후 입고된 수량 |
| 예상 하루 사용량           | 하루 평균 예상 사용량             |
| 오늘 날짜                  | 계산 기준 날짜                    |

#### 계산 흐름

1. 마지막 재고 확인일 이후 지난 일수를 계산한다.
2. 예상 하루 사용량과 지난 일수를 곱해 예상 사용량을 계산한다.
3. 마지막 확인 재고에 마지막 확인 이후 입고 수량을 더한다.
4. 더한 값에서 예상 사용량을 차감한다.
5. 계산 결과가 0보다 작으면 0으로 처리한다.

#### 계산식

현재 예상 재고: `Math.max(lastCheckedStock + receivedQuantitySinceLastChecked - expectedDailyUsage * daysSinceLastChecked, 0)`

#### 예시

| 항목                       | 값         |
| -------------------------- | ---------- |
| 마지막 확인 재고           | 30         |
| 마지막 재고 확인일         | 2026-04-25 |
| 오늘 날짜                  | 2026-04-28 |
| 예상 하루 사용량           | 5          |
| 마지막 확인 이후 입고 수량 | 0          |

지난 일수: `3일`

현재 예상 재고: `Math.max(30 + 0 - 5 * 3, 0) = 15`

### 2.6 다음 보충 주기까지 필요한 기준 재고 계산

다음 보충 주기까지 필요한 기준 재고는 다음 보충 가능일까지 남은 일수와 예상 하루 사용량, 버퍼 재고를 기준으로 계산한다.

다음 보충 가능일은 이번 입고 이후 다시 재고를 보충할 수 있는 시점을 의미한다.

#### 계산에 사용하는 값

| 항목             | 설명                                           |
| ---------------- | ---------------------------------------------- |
| 다음 보충 가능일 | 이번 입고 이후 다시 재고를 보충할 수 있는 날짜 |
| 오늘 날짜        | 계산 기준 날짜                                 |
| 예상 하루 사용량 | 하루 평균 예상 사용량                          |
| 버퍼 재고        | 예외 상황을 대비한 여유 재고                   |

#### 계산 흐름

1. 오늘부터 다음 보충 가능일까지 남은 일수를 계산한다.
2. 남은 일수와 예상 하루 사용량을 곱해 다음 보충 가능일까지 예상되는 사용량을 계산한다.
3. 예상 사용량에 버퍼 재고를 더해 필요한 기준 재고를 계산한다.

#### 계산식

다음 보충 주기까지 필요한 기준 재고: `daysUntilNextCycleRestock * expectedDailyUsage + bufferStock`

#### 예시

| 항목             | 값         |
| ---------------- | ---------- |
| 다음 보충 가능일 | 2026-05-06 |
| 오늘 날짜        | 2026-04-28 |
| 예상 하루 사용량 | 5          |
| 버퍼 재고        | 5          |

다음 보충 가능일까지 남은 일수: `8일`

다음 보충 주기까지 필요한 기준 재고: `8 * 5 + 5 = 45`

### 2.7 권장 발주 수량 계산

권장 발주 수량은 다음 보충 주기까지 필요한 기준 재고와 현재 예상 재고를 비교해 계산한다.

#### 계산에 사용하는 값

| 항목                                | 설명                                                        |
| ----------------------------------- | ----------------------------------------------------------- |
| 현재 예상 재고                      | 마지막 확인 재고, 입고 수량, 예상 사용량을 반영한 현재 재고 |
| 다음 보충 주기까지 필요한 기준 재고 | 다음 보충 가능일까지 버티기 위해 필요한 수량                |

#### 계산 흐름

1. 현재 예상 재고를 계산한다.
2. 다음 보충 주기까지 필요한 기준 재고를 계산한다.
3. 다음 보충 주기까지 필요한 기준 재고에서 현재 예상 재고를 뺀다.
4. 계산 결과가 0보다 작으면 0으로 처리한다.

#### 계산식

권장 발주 수량: `Math.max(requiredStockUntilNextCycleRestock - estimatedCurrentStock, 0)`

#### 예시

| 항목                                | 값  |
| ----------------------------------- | --- |
| 현재 예상 재고                      | 15  |
| 다음 보충 주기까지 필요한 기준 재고 | 45  |

권장 발주 수량: `Math.max(45 - 15, 0) = 30`

### 2.8 재고 상태 계산

재고 상태는 현재 예상 재고, 다음 보충 주기까지 필요한 기준 재고, 최대 재고를 기준으로 분류한다.

#### 상태 기준

| 상태 | 기준                                                                                |
| ---- | ----------------------------------------------------------------------------------- |
| 부족 | 현재 예상 재고가 다음 보충 주기까지 필요한 기준 재고보다 작은 경우                  |
| 정상 | 현재 예상 재고가 다음 보충 주기까지 필요한 기준 재고 이상이고 최대 재고 이하인 경우 |
| 과잉 | 현재 예상 재고가 최대 재고를 초과하는 경우                                          |

#### 판단 순서

1. 현재 예상 재고가 최대 재고를 초과하면 `과잉`으로 분류한다.
2. 현재 예상 재고가 다음 보충 주기까지 필요한 기준 재고보다 작으면 `부족`으로 분류한다.
3. 그 외에는 `정상`으로 분류한다.

| 예상 하루 사용량           | 0 이상    |

| 최대 재고                  | 0 이상    |

| 공급처                     | 선택 입력 |

#### 추가 검증 기준

| 조건                                              | 처리             |

### 2.12 API Mocking

백엔드 없이도 서버 상태 기반 화면을 구현하고 검증할 수 있도록 MSW로 Mock API를 구성한다.

Mock API는 실제 백엔드 구현이 아니라, 프론트엔드에서 서버 상태와 요청 / 응답 흐름을 다루기 위한 개발용 API 환경으로 사용한다.

| API                       | 설명                      |
| ------------------------- | ------------------------- |
| GET /inventory            | 재고 목록 조회            |
| POST /inventory           | 재고 품목 등록            |
| PATCH /inventory/:id      | 재고 품목 수정            |
| GET /sales/summary        | 매출 요약 조회            |
| GET /sales/trend          | 일자별 매출 추이 조회     |
| GET /sales/category-ratio | 카테고리별 판매 비중 조회 |

### 2.13 테스트

핵심 판단 로직을 UI와 분리하고, 입력값에 따라 기대한 결과가 나오는지 검증한다.

| 테스트 대상                                   | 검증 내용                                                         |
| --------------------------------------------- | ----------------------------------------------------------------- |
| 날짜 계산 함수                                | 마지막 확인일 이후 지난 일수, 다음 보충 가능일까지 남은 일수 계산 |
| 현재 예상 재고 계산 함수                      | 마지막 확인 재고, 입고 수량, 예상 사용량 반영                     |
| 다음 보충 주기까지 필요한 기준 재고 계산 함수 | 남은 일수, 예상 하루 사용량, 버퍼 재고 반영                       |
| 권장 발주 수량 계산 함수                      | 필요한 기준 재고와 현재 예상 재고 비교                            |
| 재고 상태 계산 함수                           | 부족 / 정상 / 과잉 상태 분류                                      |
| 필터링 함수                                   | 검색어와 카테고리 조건 적용                                       |
| 정렬 함수                                     | 정렬 옵션별 결과 순서                                             |
| 폼 validation schema                          | 잘못된 입력값 차단                                                |
